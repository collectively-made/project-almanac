## Pick Up Next Session

### Priority 1: Setup Flow Improvements
The setup page needs:
1. **Restore the model download/load UI** — the "Download & Start" button from the older setup page got simplified away. Bring it back so users can one-click download a model.
2. **Cloud API option** — allow users to input an Anthropic or OpenAI API key as an alternative to running a local model. Two paths:
   - **Local model** (default) — fully offline, private, but needs 8GB RAM and model download
   - **Cloud API** — instant setup, faster, smarter (Claude/GPT), but requires internet and sends queries to a third party

### Priority 2: Cloud API Key Support
Design:
- Settings page: API key input (Anthropic or OpenAI)
- Clear privacy callout when entering a key:
  - "Your queries will be sent to [Anthropic/OpenAI] servers"
  - "Your user context profile will be included in prompts"
  - "Responses may be faster and more detailed than local models"
  - "You can switch back to local mode anytime"
- Backend: abstract the LLM layer so it can route to either llama-cpp-python OR an API client
- Chat endpoint detects which mode is active and routes accordingly
- RAG pipeline stays the same — only the generation step changes
- The API key should be stored encrypted or at minimum clearly marked as sensitive

Benefits of cloud API option:
- Zero-config: no model download, works on any hardware
- Better response quality (Claude Sonnet/Opus >> 3B local model)
- Better conversational profile building
- Still uses local RAG — knowledge base stays local
- User can switch between local and cloud freely
